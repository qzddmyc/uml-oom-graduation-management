# 毕业设计管理系统重点用例实现

## 1. 用例：课题申报与审核发布

### 1.1 目标

完成课题从申报到审核、发布的全过程，支持教师盲选课题、教师指定学生课题、学生申报课题三种来源。

### 1.2 参与者

- 主参与者：指导教师、学生、专业负责人、教学院长
- 次参与者：系统管理员（特殊情况处理时）

### 1.3 前置条件

1. 相关角色账号已建立
2. 课题申报时间窗口已开启
3. 学院、专业和教师配额信息已配置

### 1.4 主成功场景

1. 指导教师或学生提交课题申报
2. 系统校验课题信息完整性和时间窗口
3. 专业负责人审核课题
4. 对需要院长发布的课题，教学院长进行发布
5. 课题进入可选或已指定状态

### 1.5 异常流程

1. 课题信息不完整，系统拒绝提交
2. 专业负责人驳回课题，申报人修改后重新提交
3. 已审课题需要修改时，进入“已审课题修改”流程

### 1.6 后置条件

课题状态变为“已发布”或“待重新提交”。

## 2. 用例：学生选题与教师确认

### 2.1 目标

实现学生对已发布课题进行选择，并由指导教师确认、专业负责人审核双选结果。

### 2.2 参与者

- 主参与者：学生、指导教师、专业负责人

### 2.3 前置条件

1. 课题已发布
2. 学生处于允许选题的时间窗口
3. 学生尚未完成最终选题确认

### 2.4 主成功场景

1. 学生浏览课题并提交选题志愿
2. 指导教师查看报名学生并确认学生
3. 系统生成双选确认结果
4. 专业负责人审核并发布双选结果

### 2.5 异常流程

1. 课题名额已满，学生需重新选择
2. 指导教师未确认，系统保持待确认状态
3. 审核不通过，返回教师或学生重新处理

### 2.6 后置条件

学生与课题建立正式关联关系。

## 3. 用例：任务书下达与审批

### 3.1 目标

实现课题确定后的任务书下达、审批和修改控制。

### 3.2 参与者

- 主参与者：指导教师、专业负责人、学生

### 3.3 前置条件

1. 学生已确定课题
2. 指导教师拥有该学生的指导关系

### 3.4 主成功场景

1. 指导教师填写并下达任务书
2. 系统保存任务书并通知专业负责人
3. 专业负责人审批任务书
4. 审批通过后，学生查看任务书

### 3.5 异常流程

1. 审批不通过，任务书退回修改
2. 已审任务书需要修改，进入任务书修改审批流程

### 3.6 后置条件

任务书状态变为“已生效”或“退回修改”。

## 4. 用例：开题报告审核与评价

### 4.1 目标

实现学生开题报告提交，并由指导教师和评价教师共同完成审核评价。

### 4.2 参与者

- 主参与者：学生、指导教师、评价教师
- 次参与者：专业负责人（查询与监督）

### 4.3 前置条件

1. 任务书已审批通过
2. 开题报告时间窗口已开启

### 4.4 主成功场景

1. 学生提交开题报告
2. 系统记录提交版本
3. 指导教师审核开题报告并给出评价
4. 评价教师参与评价
5. 系统汇总评价结果并更新状态

### 4.5 异常流程

1. 开题报告被驳回，学生修改后重新提交
2. 某评价教师未完成评价，系统保持待完成状态

### 4.6 后置条件

开题报告状态进入“通过”“待修改”或“评价中”。

## 5. 用例：中期检查评价

### 5.1 目标

实现学生提交中期检查材料，并由指导教师和评价教师进行过程评价。

### 5.2 参与者

- 主参与者：学生、指导教师、评价教师、专业负责人

### 5.3 前置条件

1. 中期检查阶段已到达
2. 学生具备中期提交资格

### 5.4 主成功场景

1. 学生提交中期检查材料
2. 指导教师填写中期评价
3. 评价教师填写补充评价
4. 专业负责人审核中期结果
5. 系统更新中期检查状态

### 5.5 异常流程

1. 中期检查被退回修改
2. 已审中期检查需要修改时，进入“中期检查修改审批”流程

### 5.6 后置条件

中期检查状态变为“已通过”“待修改”或“待复审”。

## 6. 用例：评阅教师分配

### 6.1 目标

实现专业负责人对学生论文进行评阅教师分配，并触发评阅流程。

### 6.2 参与者

- 主参与者：专业负责人、评阅教师

### 6.3 前置条件

1. 学生已提交论文定稿
2. 指导教师已完成指导评分

### 6.4 主成功场景

1. 专业负责人选择待评阅学生
2. 系统展示可选评阅教师
3. 专业负责人完成分配
4. 系统通知评阅教师进入评阅工作台

### 6.5 异常流程

1. 可用评阅教师不足，分配失败
2. 需调整评阅教师时，原分配记录进入调整流程

### 6.6 后置条件

系统生成评阅任务和待办事项。

## 7. 用例：答辩成绩录入与总评发布

### 7.1 目标

实现答辩成绩录入、综合成绩汇总和最终成绩发布。

### 7.2 参与者

- 主参与者：答辩录入员、专业负责人、教学秘书
- 次参与者：系统管理员（总评调整审核）

### 7.3 前置条件

1. 评阅成绩已完成
2. 答辩组已建立
3. 学生已进入答辩名单

### 7.4 主成功场景

1. 答辩录入员录入答辩成绩
2. 系统汇总指导教师、评阅教师、答辩成绩
3. 按权重计算总评成绩
4. 教学秘书或专业负责人发布最终成绩

### 7.5 异常流程

1. 成绩录入不完整，系统不允许发布
2. 总评成绩需要调整时，进入成绩调整申请与审核流程

### 7.6 后置条件

学生成绩状态变为“已发布”。
