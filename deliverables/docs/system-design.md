# 毕业设计管理系统技术方案、包图与构件图设计

## 1. 技术路线说明

结合课程任务要求与当前仓库规划，本项目采用如下技术路线：

| 项目层次 | 方案 |
|---|---|
| 界面原型 | Vue 3 + JavaScript |
| 数据方式 | Mock 数据 |
| 图与文档 | Markdown + PlantUML + draw.io |
| 图表分析 | ECharts |
| 原型运行 | Vite 本地开发环境 |

说明：

1. 前端原型用于展示界面设计与业务流程。
2. Mock 数据用于降低演示实现成本，但不影响数据库设计成果的完整表达。
3. 包图、构件图、部署图按“可扩展到真实系统”的思路进行设计。

## 2. 系统分层结构

系统可按以下逻辑层次进行划分：

```text
表现层
  ├── 登录与角色入口
  ├── 管理端页面
  ├── 教师端页面
  └── 学生端页面

业务控制层
  ├── 课题与选题控制
  ├── 过程材料控制
  ├── 评阅答辩控制
  ├── 成绩与统计控制
  └── 归档与特殊处理控制

数据抽象层
  ├── API 模块
  ├── Mock 模块
  └── 数据对象映射
```

## 3. 模块划分

| 模块 | 职责 |
|---|---|
| 组织与权限模块 | 学院、专业、班级、账号、角色、权限、时间节点 |
| 课题与选题模块 | 课题申报、审核、发布、双选、课题调整 |
| 过程材料模块 | 任务书、开题报告、中期检查、外文翻译、论文材料 |
| 评阅与答辩模块 | 评阅分配、评阅记录、答辩组、答辩成绩录入 |
| 成绩与统计模块 | 指导成绩、评阅成绩、答辩成绩、总评、统计分析 |
| 服务支撑模块 | 导出、归档、抽检、特殊情况处理 |

## 4. 包图设计说明

包图建议体现以下包结构：

1. `organization`：组织和权限
2. `topic`：课题与选题
3. `process`：任务书、开题、中期、论文材料
4. `evaluation`：评阅、答辩、成绩
5. `statistics`：统计、导出、归档
6. `support`：规则、通知、特殊处理

## 6. 技术方案结论

本阶段形成的技术方案强调以下几点：

1. 课程成果以建模与设计文档为核心，原型仅作为支撑展示。
2. 包图和构件图应服务于“业务清晰、职责明确、可展示可扩展”的目标。
3. 前端与 Mock 数据的实现方案不改变后续数据库设计和部署设计的完整性。
