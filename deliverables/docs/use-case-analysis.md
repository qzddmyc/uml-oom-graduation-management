# 毕业设计管理系统参与者与用例分析

## 1. 参与者识别

本系统的主要参与者如下：

| 编号 | 参与者 | 说明 |
|---|---|---|
| A1 | 系统管理员 | 负责全校级初始化、账号、规则、统计、归档与抽检 |
| A2 | 教学秘书 | 负责学院级流程管理、成绩发布、调整申请 |
| A3 | 教学院长 | 负责课题发布、学院级统计与查询 |
| A4 | 专业负责人 | 负责课题审核、双选审核、任务书审批、评阅分配、答辩分组 |
| A5 | 指导教师 | 负责课题申报、任务书下达、过程材料审核与评分 |
| A6 | 评阅教师 | 负责论文评阅和评分 |
| A7 | 答辩录入员 | 负责答辩成绩录入 |
| A8 | 学生 | 负责选题、材料提交、论文提交与各类申请 |

## 3. 核心用例清单

### 3.1 组织与规则类用例

1. 维护学院、专业、班级信息
2. 维护账号与角色
3. 设置时间节点
4. 设置评分指标与权重

### 3.2 课题与选题类用例

1. 教师申报盲选课题
2. 教师申报指定学生课题
3. 学生申报课题
4. 专业负责人审核课题
5. 教学院长发布课题
6. 学生选择课题
7. 指导教师确认学生
8. 专业负责人审核双选结果

### 3.3 过程材料类用例

1. 指导教师下达任务书
2. 专业负责人审批任务书
3. 学生提交开题报告
4. 指导教师 / 评价教师审核开题报告
5. 学生提交中期检查材料
6. 指导教师 / 评价教师评价中期检查
7. 学生提交论文初稿
8. 学生提交论文定稿
9. 指导教师审核论文与评分

### 3.4 评阅与答辩类用例

1. 专业负责人分配评阅教师
2. 评阅教师评阅论文
3. 专业负责人创建答辩组
4. 答辩录入员录入答辩成绩
5. 教学秘书或专业负责人发布总评成绩

### 3.5 管理与服务类用例

1. 调整课题
2. 审批任务书修改
3. 审批中期检查修改
4. 处理总评调整申请
5. 查询统计信息
6. 导出表格
7. 查询历史归档

## 4. 用例分组结论

从分析结果看，系统可划分为以下主要用例域：

1. **基础管理域**：组织、账号、角色、时间节点、评分规则
2. **课题选题域**：课题申报、审核、发布、双选
3. **过程材料域**：任务书、开题、中期、论文
4. **评阅答辩域**：评阅分配、评阅评分、答辩分组、答辩录入
5. **成绩与服务域**：总评、调整、统计、导出、归档
