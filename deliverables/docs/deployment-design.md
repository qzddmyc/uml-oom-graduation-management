# 毕业设计管理系统部署设计与运行环境说明

## 1. 设计目标

部署设计用于说明系统从逻辑架构到运行节点的部署方式，满足课程要求中的“部署图设计”成果。

## 2. 部署设计原则

1. 演示环境与完整系统逻辑部署分开说明
2. 浏览器端、应用层、数据层的职责边界清晰
3. 即使当前原型使用 Mock 数据，也保留可扩展到真实后端与数据库的部署结构

## 3. 原型演示环境

当前原型的实际运行方式如下：

| 节点 | 说明 |
|---|---|
| 浏览器 | 访问前端原型页面 |
| 前端应用 | 由 Vue 3 + Vite 提供页面渲染 |
| Mock 数据模块 | 提供本地数据与接口模拟 |

该环境适合课程演示、页面原型展示与流程说明。

## 4. 逻辑完整部署方案

若系统进一步落地，可按以下逻辑部署：

| 层次 | 节点 | 说明 |
|---|---|---|
| 表现层 | Web 浏览器 | 学生、教师、管理人员访问入口 |
| 接入层 | Nginx / 静态资源服务 | 提供前端页面资源 |
| 应用层 | 业务应用服务 | 提供课题、过程、评阅、成绩、统计等接口 |
| 数据层 | MySQL 数据库 | 存储组织、课题、材料、评阅、答辩、成绩与归档数据 |
| 文件层 | 文件存储服务 | 存储论文材料、导出文件、附件 |

## 5. 运行环境说明

### 5.1 原型运行环境

| 项目 | 说明 |
|---|---|
| 前端运行方式 | Vite 本地开发环境 |
| 数据来源 | Mock 数据 |
| 文档编写 | Markdown |
| UML 绘图 | PlantUML |
| 补充绘图 | draw.io |

### 5.2 逻辑系统运行环境

| 项目 | 说明 |
|---|---|
| 客户端 | 现代浏览器 |
| 前端部署 | 静态资源服务器 |
| 业务服务 | 可扩展的服务端应用 |
| 数据存储 | 关系型数据库 |
| 附件存储 | 文件服务 |

## 7. 结论

部署设计需要同时满足两个层次：

1. **课程演示层**：原型可以运行、可以展示业务流程。
2. **系统设计层**：部署图要体现完整系统的浏览器、应用服务、数据库与文件服务关系。
