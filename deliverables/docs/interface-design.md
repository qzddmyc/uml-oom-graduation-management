# 毕业设计管理系统界面设计说明

## 1. 设计目标

界面设计用于展示毕业设计管理系统在课程作业中的原型方案，使需求分析、用例模型和系统设计能够通过可视化页面进一步落地。界面设计的目标包括：

1. 直观呈现多角色参与的业务流程。
2. 体现课题、过程材料、评阅答辩、成绩与归档等核心模块。
3. 用统一布局和交互模式降低业务理解成本。
4. 为最终报告截图、PPT 展示和录屏演示提供稳定素材来源。

## 2. 设计原则

1. **角色优先**：页面组织优先围绕系统管理员、教学秘书、专业负责人、指导教师、评阅教师、答辩录入员、学生等角色展开。
2. **流程优先**：导航结构围绕课题、任务书、开题、中期、论文、评阅、答辩、总评等阶段设计。
3. **状态可见**：重要业务对象需通过标签、步骤、时间节点、待办列表等方式呈现当前状态。
4. **信息分层**：仪表盘用于概览，列表页用于批量处理，详情/表单页用于材料提交、审核和评价。
5. **统一样式**：登录页、首页、业务页和工作台采用统一品牌色、卡片风格、表格交互和消息提醒机制。

## 3. 信息架构与导航设计

### 3.1 顶层导航结构

前端原型采用统一工作台布局，主要页面分组如下：

| 分组 | 主要页面 | 说明 |
|---|---|---|
| 公共入口 | 登录、角色切换、403、404 | 承担访问入口与异常状态展示 |
| 通用工作台 | 仪表盘、个人中心、消息中心 | 展示待办、提醒、个人资料和通知 |
| 课题与选题 | 课题列表、课题申报、课题审核、课题发布、学生选题、教师确认、双选审核 | 覆盖课题从申报到确定的全过程 |
| 过程材料 | 任务书详情/审批、开题提交/审核、中期提交/审核、论文提交/审核 | 覆盖材料提交、审核、评价与修改 |
| 评阅与答辩 | 评阅分配、评阅工作台、答辩分组、答辩成绩录入 | 体现后期过程组织和分工 |
| 成绩与统计 | 成绩汇总、成绩发布、成绩调整、统计分析 | 体现总评、发布和宏观分析 |
| 归档与系统管理 | 归档查询、导出中心、用户管理、组织管理、时间节点、评分规则 | 展示后台支撑能力 |

### 3.2 路由与权限映射

界面原型的路由与角色权限来源于 `plans/frontend-page-plan.md` 中的页面清单与路由设计。页面访问控制主要通过角色过滤实现，按钮级能力通过业务语义体现，例如“审核”“发布”“分配”“导出”“调整”等操作。

## 4. 统一布局与交互说明

### 4.1 布局结构

统一布局主要包括以下区域：

1. 顶部导航：展示系统名称、角色切换入口、用户信息和快捷操作。
2. 侧边菜单：按业务域组织菜单项，并根据角色动态过滤。
3. 面包屑与标题区：展示当前业务位置和页面定位。
4. 内容区：承载卡片、表单、时间线、表格、统计图等页面主体。
5. 消息与状态反馈：通过标签、提示、进度条、通知等方式反馈操作状态。

### 4.2 通用交互模式

| 交互模式 | 适用页面 | 设计意图 |
|---|---|---|
| 筛选栏 + 表格卡片 | 列表页、管理页 | 方便批量查询、审核、导出 |
| 信息面板 + 时间线 | 仪表盘、个人中心 | 强化阶段状态与待办提醒 |
| 表单 + 步骤说明 | 提交页、审核页 | 降低表单操作复杂度 |
| 统计卡片 + 图表 | 仪表盘、统计页 | 强化宏观展示能力 |
| 抽屉 / 弹窗 / 标签 | 详情查看、状态反馈 | 减少页面跳转并突出重点信息 |

## 5. 角色化页面设计

### 5.1 学生端

学生端重点围绕“选题 - 材料提交 - 过程跟踪 - 成绩查询”组织，核心页面包括：

1. 学生选题页
2. 任务书查看页
3. 开题报告提交页
4. 中期检查提交页
5. 论文初稿 / 定稿提交页
6. 成绩查询与消息提醒页

### 5.2 教师端

教师端重点围绕“课题管理 - 任务书下达 - 过程审核 - 论文审核”组织，核心页面包括：

1. 课题申报页
2. 教师确认页
3. 开题审核页
4. 中期评价页
5. 论文审核页

### 5.3 管理端

管理端重点围绕“审核、分配、分组、发布、统计、归档”组织，核心页面包括：

1. 课题审核页
2. 双选审核页
3. 任务书审批页
4. 评阅教师分配页
5. 答辩分组管理页
6. 答辩成绩录入页
7. 成绩汇总与发布页
8. 统计分析页
9. 归档查询 / 导出页
10. 用户与时间节点设置页

## 9. 结论

界面设计说明的作用不是替代 UML 建模或系统设计，而是将抽象的业务流程、角色职责和状态变化转化为可展示的页面结构。通过统一布局、角色化导航和流程化页面设计，前端原型能够为课程汇报和最终报告提供直观支撑。
