# 毕业设计管理系统录屏演示脚本

## 1. 演示目标

在 5~10 分钟内，结合 PPT 和前端原型，展示项目背景、核心建模成果和界面原型效果。

## 2. 推荐时长分配

| 阶段 | 建议时长 | 说明 |
|---|---|---|
| 背景与问题 | 0.5~1 分钟 | 说明为什么要重设计系统 |
| 需求与 UML 建模 | 2~3 分钟 | 展示用例、顺序图、类图、活动图、状态图 |
| 系统设计 | 1.5~2 分钟 | 展示数据库、包图、构件图、部署图 |
| 前端原型演示 | 2~3 分钟 | 展示关键页面和流程 |
| 总结 | 0.5~1 分钟 | 强调课程要求覆盖与项目亮点 |

## 3. 具体脚本

### 3.1 开场

“大家好，我们小组本次面向对象建模课程综合设计的题目是毕业设计管理系统。项目目标是围绕现有系统在过程评价、评价主体、复杂操作和统计分析方面的不足，重新完成一套完整的分析与设计方案。”

### 3.2 需求与问题分析

1. 展示项目背景和现有问题。
2. 说明需求获取方式：文档分析、场景化访谈演练、关键流程推演。
3. 强调系统需要支持多角色、全过程、强状态的管理特征。

### 3.3 UML 建模成果展示

1. 展示系统总用例图，说明主要参与者。
2. 展示 1~2 张分角色用例图，说明角色职责划分。
3. 展示重点用例实现与顺序图，重点讲课题审核、开题评价或成绩发布。
4. 展示类图，说明课题、过程材料、评阅答辩、成绩单等核心对象。
5. 展示活动图和状态图，强调流程控制和状态流转。

### 3.4 系统设计成果展示

1. 展示技术路线，说明前端原型与建模文档的关系。
2. 展示数据库 ER 图，说明核心数据表和成绩计算逻辑。
3. 展示包图和构件图，说明模块划分。
4. 展示部署图，说明课程演示环境和完整系统部署思路。

### 3.5 前端原型展示

推荐演示顺序如下：

1. 登录页
2. 首页仪表盘
3. 课题列表 / 学生选题
4. 开题审核或中期评价页
5. 评阅教师分配或答辩分组页
6. 成绩汇总页
7. 统计分析页

### 3.6 结束语

“以上就是我们围绕毕业设计管理系统完成的需求分析、UML 建模、系统设计和界面原型成果。谢谢大家。”

## 4. 录屏注意事项

1. 录屏前提前打开需要展示的文档、图片和前端页面。
2. 页面切换顺序尽量固定，减少现场查找。
3. 重点内容尽量配合高亮或放大后的图片展示。
4. 若最终录屏文件由组长统一整合，可按本脚本拆分角色讲解部分。
