# 毕业设计管理系统粗略实现规划

## 1. 项目目标

本项目的目标是依据 `docs/Requirement.md` 完成面向对象建模课程所要求的系统分析与设计成果，并在必要时提供可演示的前端原型。规划重点如下：

- 补充开题评价、中期评价等过程控制环节
- 增加评价教师，完善评价主体
- 优化评阅、答辩分组等复杂操作
- 增强统计分析能力

## 2. 需完成的内容

依据 `Requirement.md`，需完成以下成果：

1. 需求获取方案（调研计划或访谈计划）
2. 用例模型
3. 用例实现
4. 类图
5. 顺序图
6. 活动图和状态图（按需）
7. 开发工具与开发框架方案
8. 数据库设计
9. 包图和构件图
10. 部署图
11. 程序界面设计
12. 设计报告
13. 汇报 PPT 与录屏材料

## 3. 核心业务事件

分析设计阶段应覆盖以下核心业务事件：

1. 用户与角色初始化
2. 课题申报、审核与发布
3. 学生选题或学生自拟课题申报
4. 任务书下达与确认
5. 开题报告提交、审核与评价
6. 中期检查提交、审核与评价
7. 论文过程材料提交与修改
8. 评阅教师分配与论文评阅
9. 答辩分组、答辩录入与成绩汇总
10. 总评发布
11. 统计分析、导出与历史归档
12. 特殊情况处理

## 4. 总体实现方案

### 4.1 需求与范围澄清

- 明确系统边界、目标角色和主要问题
- 输出调研计划或访谈计划
- 提炼角色、业务对象和主流程

### 4.2 需求分析建模

- 识别参与者与用例
- 绘制总体用例图
- 编写重点用例描述
- 提炼主成功场景与异常场景

### 4.3 静态结构设计

- 识别用户、课题、任务书、开题、中期、论文、评阅、答辩、成绩、统计等核心实体
- 明确实体之间的关联、职责和边界
- 输出类图与包图

### 4.4 动态行为设计

- 为关键流程绘制顺序图
- 为审批流绘制活动图
- 为关键对象绘制状态图

### 4.5 数据与架构设计

- 输出数据库设计结果，用于课程文档交付
- 明确系统模块划分、构件关系和部署方式
- 原型演示部分使用 mock 数据，不接真实数据库

### 4.6 界面与交付物整理

- 设计主要角色界面原型
- 整理设计报告结构
- 准备汇报 PPT 与演示素材
- 当前 `frontend/` 中已落地 Vue 3 + Mock 数据原型，后续重点转为截图整理、展示说明和汇报材料配套

## 5. 技术栈与工具

| 层次 | 方案 |
|---|---|
| 前端原型 | Vue 3 + JavaScript |
| 数据方式 | Mock 数据 |
| 数据库 | 不接真实数据库，数据库设计仅作为课程成果输出 |
| 图与文档 | PlantUML + draw.io + Markdown |
