# 毕业设计管理系统设计报告大纲

## 1. 报告整体结构

1. 封面
2. 项目背景与需求分析
3. 参与者与用例模型

## 2. 章节与仓库文件对照

| 报告章节 | 主要内容 | 对应仓库文件 |
|---|---|---|
| 项目背景与需求分析 | 问题来源、需求获取方案、实施结果、业务范围 | `deliverables/docs/requirements-analysis.md` |
| 参与者与用例模型 | 参与者识别、职责矩阵、核心用例、用例图 | `deliverables/docs/use-case-analysis.md` |
| 重点用例实现与顺序图 | 重点用例、主成功场景、异常流程、顺序图 | `deliverables/docs/use-case-realizations.md` |
| 领域分析与类图 | 核心对象、类分类、属性与关系、类图 | `deliverables/docs/domain-class-design.md` |
| 活动图与状态图 | 关键流程活动图、对象状态图 | `INDEX.md` 中 8.4 图索引 + `deliverables/uml/` / `deliverables/render/` |
| 系统设计 | 技术路线、界面设计、数据库、包图、构件图、部署 | `deliverables/docs/system-design.md`、`deliverables/docs/interface-design.md`、`deliverables/docs/database-design.md`、`deliverables/docs/deployment-design.md` |
| 总结 | 项目成果与系统改进价值 | 可在最终导出前整理 |
| 小组成员与个人工作说明 | 小组信息、个人贡献 | 待补充真实信息 |

## 3. 建议写作顺序

1. 先以 `requirements-analysis.md` 形成背景与需求分析章节。
2. 再将用例分析、用例实现、类图设计组织为 UML 建模章节主体。
3. 最后整理系统设计章节，纳入界面设计、数据库、包图、构件图和部署设计。
4. 在最终导出前补充总结、小组成员和个人工作说明。
