# 毕业设计管理系统汇报 PPT 详细规划

## 1. 汇报目标

本 PPT 用于支撑面向对象建模课程综合设计汇报，重点是把“问题来源、需求分析、UML 建模、系统设计、界面原型、最终价值”按清晰顺序展示出来。建议总页数控制在 **14 页左右**，汇报时长控制在 **6~8 分钟**，既能覆盖课程要求，也便于后续录屏。

## 2. 汇报结构总览

1. 封面
2. 项目背景与问题来源
3. 需求获取与分析结论
4. 参与者与角色职责
5. 系统总用例与分角色用例
6. 重点用例实现
7. 顺序图展示
8. 活动图与状态图展示
9. 领域类图与分析类图
10. 数据库设计
11. 技术路线、包图与构件图
12. 部署设计
13. 界面设计与前端原型
14. 成果总结与后续待完成项

## 3. 逐页详细规划

### 第 1 页：封面

**页面标题**

- 面向对象建模综合设计
- 毕业设计管理系统

**页面目标**

- 交代课程名称、项目名称和汇报身份。
- 让听众快速进入“毕业设计管理系统”这一主题。

**建议内容**

1. 课程名称
2. 项目名称
3. 小组名称 / 组号
4. 成员姓名（如果暂未确定，可先留占位）
5. 汇报日期

**建议版式**

- 左侧放标题与课程信息
- 右侧可放系统关键词或简洁插图
- 整体保持简洁，不要堆太多文字

**讲解要点**

- “我们本次综合设计的题目是毕业设计管理系统。”
- “本汇报将围绕需求分析、UML 建模、系统设计与界面原型四个部分展开。”

**建议时长**

- 15~20 秒

---

### 第 2 页：项目背景与问题来源

**页面标题**

- 项目背景与问题来源

**页面目标**

- 说明为什么要做这个题目。
- 把课程给定背景和现有系统痛点交代清楚。

**建议内容**

1. 毕业设计是本科阶段的重要实践环节
2. 管理过程涉及课题、任务书、开题、中期、论文、评阅、答辩、总评等多个阶段
3. 现有系统存在的主要问题：
   - 过程评价环节不足
   - 评价主体单一
   - 评阅和答辩组织复杂
   - 缺少统计分析能力

**建议素材**

- 可直接提炼 `deliverables/docs/requirements-analysis.md` 中“项目背景与建设目标”“问题分析”部分

**讲解要点**

- 强调“不是从零虚构业务，而是针对已有系统问题进行重设计”
- 强调“问题驱动后续建模”

**建议时长**

- 30~40 秒

### 第 3 页：需求获取与分析结论

**页面标题**

- 需求获取方法与分析结论

**页面目标**

- 展示需求不是拍脑袋来的，而是有获取方式和分析过程。
- 引出后续 UML 建模依据。

**建议内容**

1. 需求获取方法：
   - 文档分析
   - 场景化访谈演练
   - 关键流程推演
2. 主要参与对象：
   - 管理角色
   - 教师角色
   - 学生角色
3. 分析结论：
   - 系统是多角色、强流程、强状态业务系统
   - 必须体现多主体评价和过程控制
   - 统计、导出、归档是重要支撑能力

**建议素材**

- `deliverables/docs/requirements-analysis.md`

**讲解要点**

- 不需要细讲所有访谈问题，重点讲“得出了什么结论”
- 这一页是后续用例和类图的逻辑起点

**建议时长**

- 30~40 秒

**建议素材**

- `deliverables/docs/use-case-analysis.md` 中的参与者识别与角色职责矩阵

**建议版式**

**讲解要点**

- 强调“不同角色围绕同一主流程分工协作”

**建议时长**

## 4. 页面设计建议

1. **一页只讲一个中心问题**，不要同时塞太多图。
2. **文字少而精**，每页控制在 3~5 个核心点。
3. **图片优先使用 render 目录中的 PNG**，避免现场打开 `.puml`。
4. **界面展示页尽量使用真实截图**，不要只贴页面列表。
5. **总结页不要重复前文**，只做归纳收束。

## 5. 建议准备的素材清单

1. 用例图 PNG
2. 顺序图 PNG
3. 活动图与状态图 PNG
4. 类图 PNG
5. ER 图 PNG
6. 包图 / 构件图 / 部署图 PNG
7. 登录页、仪表盘、学生选题、审核页、成绩页、统计页截图
8. 最终组员信息和汇报封面信息

## 7. 录屏讲解脚本

### 7.1 开场

“大家好，我们小组本次面向对象建模课程综合设计的题目是毕业设计管理系统。项目目标是围绕现有系统在过程评价、评价主体、复杂操作和统计分析方面的不足，重新完成一套完整的分析与设计方案。”

### 7.2 需求与问题分析部分

1. 先说明毕业设计管理涉及课题、任务书、开题、中期、论文、评阅、答辩和总评等多个阶段。
2. 再指出现有系统的四类问题：过程评价不足、评价主体单一、复杂操作较多、统计分析不足。
3. 接着说明需求获取方式来源于文档分析、场景化访谈演练和关键流程推演。

### 7.3 UML 建模成果部分

1. 先展示系统总用例图，说明系统覆盖范围和角色集合。
2. 再展示分角色用例图，说明管理侧、教师侧和学生侧职责。
3. 之后重点讲 2~3 个关键用例，例如课题审核、开题评价、成绩发布。
4. 展示顺序图时，重点说明多角色之间的交互顺序。
5. 展示类图、活动图和状态图时，强调对象关系、流程控制和状态流转。

### 7.4 系统设计成果部分

1. 说明课程展示阶段采用 Vue 3、Mock 数据、Markdown、PlantUML 和 draw.io。
2. 展示数据库 ER 图，说明核心数据结构。
3. 展示包图和构件图，说明模块划分。
4. 展示部署图，区分当前演示环境和完整系统部署思路。

### 7.5 前端原型展示部分

建议演示顺序如下：

1. 登录页
2. 首页仪表盘
3. 课题列表 / 学生选题
4. 开题审核或中期评价页
5. 评阅教师分配或答辩分组页
6. 成绩汇总页
7. 统计分析页

### 7.6 结束语

“以上就是我们围绕毕业设计管理系统完成的需求分析、UML 建模、系统设计和界面原型成果。谢谢大家。”

### 7.7 录屏注意事项

1. 录屏前提前打开需要展示的文档、图片和前端页面。
2. 页面切换顺序尽量固定，减少现场查找。
3. 重点内容尽量配合高亮或放大后的图片展示。
4. 如果最终录屏由组长统一整合，可按本脚本拆分不同成员的讲解部分。

## 8. 结论

这份规划可以直接作为后续制作正式 PPT 的蓝本使用。下一步如果需要继续推进，可以在此基础上进一步生成“每一页的正式文案版本”或“可直接复制到 PPT 的简化版标题与要点”。
