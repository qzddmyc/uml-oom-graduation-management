# 面向对象建模综合设计

课程目标：通过本课程的教学，学生能够熟练地使用UML2表达设计思想，运用对象技术的原则和方法完成应用系统的分析和设计过程，并对面向对象思想和理论有进一步的理解。

综合设计目标：通过整个实践过程，掌握课程中所介绍的原理和方法，熟练运用相关软件工程方法和工具，来完成整个系统的分析设计工作。

项目需求简介：

毕业设计是学生大学阶段的综合训练教学环节，是培养学生工程实践能力、理论研究能力和创新意识的重要环节，是学生毕业及学位资格认定的重要依据。实施毕业设计的过程管理是提高毕业设计质量、培养合格本科毕业生的有效措施。某大学根据毕业设计要求制定了毕业设计大纲（见附件1），规定了毕业设计的工作、步骤和流程。并且使用了XXXX毕业设计管理系统（见附件2），在使用毕业设计系统的过程中发现，现有系统还存在一些问题，如：

1.  还缺少一些过程控制环节评价环节，如开题评价、中期评价

2.  开题、中期检查等只有指导教师评价，评价人比较单一，需要增加评价教师

3.  系统使用操作比较复杂，如评阅、答辩分组工作量较大

4.  缺少宏观统计分析功能

5.  其他

现需要重新设计开发一套更加完善的毕业设计管理系统，使得现有毕业设计系统更加完善。

要求：

1.  按照使用面向对象建模技术进行系统分析设计，并形成设计文档。

2.  具体要求如下：

> （1）设计需求获取方案并实施（调研计划或者访谈计划），设计用例模型
>
> （2）构造用用例实现，设计类图
>
> （3）根据用例和软件设计方法设计顺序图
>
> （4）根据需要绘制活动图和状态图
>
> （5）选择合适的开发工具和开发框架，设计数据库、设计构件图和包图。
>
> （7）设计程序界面（建议使用建模或者AI工具）
>
> （6）设计部署图

3.设计分组进行（不超4人），组长负责分工和工作协调。

4.每人提交设计报告，需明确注明小组人员、组长、自己所做的工作。

5.每组提交一份汇报ppt，并录屏，文件大小出超过200M，时间5\~10分钟，提交文件名：建模汇报X组.mp4，如建模汇报01组.mp4。

报告文件名 学号+姓名.pdf

报告提交截止时间：6.9日
